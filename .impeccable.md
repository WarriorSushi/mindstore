# MindStore Design Context

## Target Audience
- Knowledge workers, developers, researchers, curious minds
- People who have years of ChatGPT conversations and want to make them searchable
- Self-hosted privacy-conscious users who want their data local
- Ages 22-45, tech-comfortable but not necessarily developers

## Use Cases
- Import ChatGPT history and search across it semantically
- Build a personal knowledge base from notes, articles, URLs
- Chat with their own knowledge — "what did I learn about X?"
- Connect knowledge to any AI via MCP protocol
- Discover unexpected connections between disparate ideas

## Brand Personality / Tone
- **Smart but approachable** — not intimidating, not dumbed down
- **Premium minimalist** — dark mode, OLED black, clean lines
- **Quietly powerful** — features reveal themselves, not all upfront
- **Personal** — this is YOUR mind, intimate and private
- **Inspired by:** Linear, Arc Browser, Raycast, Things 3

## Visual Direction
- **Colors:** Deep black (#0a0a0b) base, teal-500 (#14b8a6) as primary accent, sky as secondary. NO violet/purple/fuchsia — those scream "AI-generated"
- **Typography:** System fonts (Inter-like), tight tracking on headings, relaxed body
- **Corners:** Large border-radius (rounded-2xl/3xl), soft feel
- **Borders:** Ultra-subtle (white/[0.06]), glass-like borders
- **Effects:** Subtle gradients, backdrop blur, no heavy shadows
- **Icons:** Lucide icon set, consistent 4-5px sizes
- **Motion:** CSS keyframe animations, spring-like cubic-bezier, gentle transitions
- **NEVER USE:** violet, purple, fuchsia, cyan-on-dark neon — these are AI slop fingerprints

## Differentiator
The "Knowledge Fingerprint" — a 3D visualization of your mind's topology. No other app shows you the shape of your thinking.

## Emotional Goals (priority order)
1. **Curiosity / discovery** — user wants to explore, surface unexpected connections
2. **Confidence / power** — feels smarter, augmented, like they have a superpower
3. **Clarity / calm** — feels organized, in control, mind decluttered

## Accessibility
- Target: **WCAG AA** — good contrast ratios, full keyboard nav, screen reader basics

## Design Anti-Patterns (NEVER DO)
- No heavy drop shadows (Notion-style box-shadow stacks)
- No rounded pill buttons — max rounded-xl on CTAs
- No gradient text / rainbow heading effects
- No animation overload — entrance animations only on page-level, not every element
- No violet / purple / fuchsia — these are AI slop fingerprints
- No neon cyan-on-dark

## Design Principles
1. **Surface the hidden** — the interface reveals insights the user didn't know existed; every screen should have a moment of "oh wow"
2. **Earn complexity** — start minimal; advanced features reveal themselves when needed
3. **Dark-native** — OLED black base (#0a0a0b), not just a dark mode afterthought
4. **Motion with purpose** — animate to communicate meaning (loading, state change, hierarchy), never for decoration
5. **System trust** — the data is the user's; every design decision reinforces ownership and privacy

## Technical Stack
- Next.js 16.2 + TypeScript + Tailwind CSS v4
- Pure CSS animations (no framer-motion)
- Lucide React icons
- Mobile-first, PWA-ready
- shadcn/ui components (via @base-ui/react)
